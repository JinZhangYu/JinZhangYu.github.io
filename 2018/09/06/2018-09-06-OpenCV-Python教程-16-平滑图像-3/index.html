1 导向滤波（Guided Filter)

引导滤波是由 何凯明等人于2010年发表在ECCV的文章《Guided Image Filtering》中提出的，后续于2013年发表了 改进算法快速引导滤波的实现。

  • 引导滤波（导向滤波）是一种图像滤波技术，通过一张 引导图，对初始图像p（输入图像）进行滤波处理，使得 最后的输出图像大体上与初始图像P 相似，但是 纹理部分与引导图相似。其典型应用有两个：保边图像平滑，抠图。
  • 引导滤波（导向滤波）的目的是，保持双边滤波的优势（有效保持边缘，非迭代计算） ，而克服双边滤波的缺点（设计一种时间复杂度为 O(1) 的快速滤波器，而且在主要边缘附近没有梯度的变形）。
  • 引导滤波（导向滤波）不仅能实现双边滤波的边缘平滑，而且 在检测到边缘附近有很好的表现，可应用在图像增强、HDR压缩、图像抠图及图像去雾等场景。

1.2 Theory of Guided Filter

1.2.1 各向同性滤波

从最简单的情况开始，假设有一个原始图像 $p$，其中含有一些噪声，欲将这些噪声滤出，得到过滤后的图片 $q$。就像我们前几章讲的那样，我们一般用的是卷积操作，



具体而言，对于均值滤波和高斯滤波中，有如下公式成立$$
q_i=\sum_{j\in w_i}W_{i,j}\cdot p_j
$$其中 $W_{i,j}$ 在均值滤波中为 $1/n$, $n$ 是窗口 $w$ 中的所有像素个数。也就是说，在均值滤波中，以像素 $i$ 为中心的一个窗口 $w$ 中的像素具有等同的权值。但是在高斯平滑中，权值 $W_{i,j}$ 将服从二维的高斯分布，结果导致离像素 $i$ 更接近的像素将具有更高的权重，反之离像素 $i$ 较远的像素则具有更小的权重。

无论是均值滤波，还是高斯滤波，它们都有一个共同的弱点，即它们都属于各向同性滤波。我们都知道，一幅自然的图像可以被看成是有（过渡平缓的，也就是梯度较小）区域和（过渡尖锐的，也就是梯度较大）边缘（也包括图像的纹理、细节等）共同组成的。噪声是影响图像质量的不利因素，我们希望将其滤除。噪声的特点通常是以其为中心的各个方向上梯度都较大而且相差不多。边缘则不同，边缘相比于区域也会出现梯度的越变，但是边缘只有在其法向方向上才会出现较大的梯度，而在切向方向上梯度较小。

因此，对于各向同性滤波（例如简单平滑或高斯平滑）而言，它们对待噪声和边缘信息都采取一直的态度。结果，噪声被磨平的同时，图像中具有重要地位的边缘、纹理和细节也同时被抹平了。这是我们所不希望看到的。

1.2.2 导向滤波与非各向同性滤波

现在我们看看导向滤波，导向滤波之所以叫这个名字，因为在算法框架中，要对 $p$ 进行滤波而得到 $q$，还得需要一个引导图像 $I$。此时，滤波器的数学公式为$$
q_i=\sum_{j\in w_i}W_{i,j}(I)\cdot p_j
$$注意，这里的 $W_{i,j}(I)$ 就表示由引导图像 $I$ 来确定加权平均运算中所采用的权值。注意，引导图像可以是单独的一幅图像，也可以输入的图像 $p$ 本身。当然为了省事我们一般选择 原彩色图的灰度图作为导引图 $I$ , 或者 对灰度图进行一些保留边缘的滤波操作再作为导引图。

导向滤波滤波的示意图如下所示，上侧部分意味每个原图的像素点 $p_i$ 都带有噪声 $n_i$，只要通过 $p_i-n_i$ 即可得到理想图片的对应像素 $q_i$。可惜的是我们无法得知任何一个具体的 $n_i$ 的值。



所以我们尝试通过下侧的部分从导引图计算出想要的图。注意这也是导向滤波所依赖的一个重要假设——导向滤波器在导向图像 $I$ 和滤波输出 $q$ 之间在一个二维窗口内是一个局部线性模型，$a$ 和 $b$ 是当窗口中心位于 $k$ 时该线性函数的系数，即 $$
q_i=a_k I_i+b_k,\forall i\in w_k
$$其中 $a_k,b_k$ 表示这两个函数只视窗口而定，窗口不一样则这两个值不一样，窗口一样则这两个值一样。而且 $a_k,b_k$ 是关于原图 $p$ 和导向图 $I$ 的函数。

导引图像与 $q$ 之间存在线性关系，这样设定是因为我们希望导引图像提供的是信息主要用于指示哪些是边缘哪些是区域，所以在滤波时，如果导引图告诉我们这里是区域，那么就将其磨平。如果导引图告诉我们这里是边缘，这在最终的滤波结果里就要设法保留这些边缘信息。

而且 只有当 $I$ 和 $q$ 之间是线性关系的这种引导关系才有意义。通过求梯度运算可以看到 $\nabla q= a\nabla I$ ,当导向图存在edge时，输出图也保留了edge。可以看到这里一定要用线性模型才能保留相应的匹配关系。

当然你也可以想象一种情况来理解这种假设。$I$ 中的甲处和乙处都是区域，而丙处是边缘。那甲处的梯度和乙处的梯度应该不会相差太大，例如乙处的梯度是甲处梯度的1.5倍。但是由于丙处是边缘，所以丙处的梯度会比较大，例如可能是甲处的6倍，也就是乙处的4倍。如果 $I$ 和 $q$ 之间满足线性关系，那么甲乙丙的梯度大小和倍数关系就都不会被扭曲。否则，如果二者之间的关系是非线性的，那么可能的结果是在 $q$ 中，尽管丙处的梯度仍然大于乙处的梯度，进而大于甲处的梯度，但是倍数关系可能会扭曲。例如，丙处的梯度是乙处的1.5倍，而是甲处的6倍（即乙处的梯度是甲处的4倍），这时你就会想象，甲处是区域，而乙处和丙处就变成了边缘。可见 非线性关系会使得引导图像对于边缘和区域的指示作用发生错乱。

1.2.3 导向滤波与岭回归

由于我们已经看到现在已知的是 $I$ 和 $p$，要求的是 $q$。而如果能求得参数 $a$ 和 $b$，显然就能通过 $I$ 和 $q$ 之间的线性关系来求出 $q$。另一方面，还可以知道 $p$ 是 $q$ 受到噪声污染而产生的退化图像，假设噪声是 $n$，则有 $q_i=p_i-n_i$。

这里就涉及到无约束图像复原的方法了，然而我并没有学过，估计会在之后的博客讲一下，不过也可以查看 约束复原与维纳滤波 作为参考，或者直接接受接下来的结论。

根据无约束图像复原的方法，这时可以设定最优化目标为 $min||n||$，等价地有 $min\text{ }n^2$，即 $min\sum_{i\in w_k}(q_i-p_i)^2$。即一个中心在 $k$ 的窗口 $w_k$ 的 cost function 有,$$
E(a_k,b_k) = \sum_{i\in w_k}(a_k I_i + b_k-p_i)^2
$$这就变成一个最小二乘问题，但是我们也知道普通最小二乘有时候引起一些麻烦，因此要适当地引入惩罚项，即采用正则化的手段。原作者在处理这里时所采用的方法借鉴了从普通线性回归改进到岭回归时所采用的方法，即 cost function 改为$$
E(a_k,b_k) = \sum_{i\in w_k}[(a_k I_i + b_k-p_i)^2 + \epsilon a_k^2]
$$求解上述最优化问题（跟最小二乘法的推导过程一致），便会得到：$$
\begin{cases}
a_k = \dfrac{\frac{1}{|w|}\sum_{i\in w_k}I_ip_i - \mu_i \overline{p_k}}{\sigma^2_k+\epsilon}\\
b_k = \overline{p_k} - a_k \mu_k
\end{cases}
$$其中，$μ_k$ 是 $I$ 中窗口 $w_k$ 中的平均值，$σ^2_k$ 是 $I$ 中窗口 $w_k$ 中的方差，$|w|$ 是窗口 $w_k$ 中像素的数量，$\overline{p_k}$ 是待滤波图像 $p$ 在窗口 $w_k$ 中的均值，即 $\overline{p_k}=\frac{1}{|w|}\sum_{i\in w_k}p_i$。

其实这个推导我完全不会，具体可以看 暗通道先验之Guided filter导向滤波

1.2.4 导向滤波与局部线性模型

在引导滤波的定义中，用到了局部线性模型，至于该模型，可以暂时用下图简单的理解：



该模型认为，某函数上一点与其邻近部分的点成线性关系，一个复杂的函数就可以用很多局部的线性函数来表示，当需要求该函数上某一点的值时，只需计算所有包含该点的线性函数的值并做平均即可。这种模型，在表示非解析函数上，非常有用。

当我们在计算像素点 $i$ 的输出 $q_i$ 时，有如下公式$$
q_i=a_k I_i+b_k,\forall i\in w_k
$$其中 $w_k$ 代表中心在 $k$ 的一个小窗口但是却包含了点 $i$，而且 $a_k,b_k$ 也是关于窗口的函数。

那现在问题来了，对于任何一个像素点 $i$，会有多个窗口可以覆盖到它，而且这些窗口的 $a,b$ 都不尽相同。也就是说，每个像素都由多个线性函数所描述。因此，如之前所说，要具体求某一点的输出值时，只需将所有包含该点的线性函数值平均即可，如下 $$
q_i = \frac{1}{|w|}\sum_{k:i\in w_k} (a_k I_i + b_k) = \overline{a_i}I_i+\overline{b_i}
$$其中 $k:i\in w_k$ 意味所有包含了点 $i$ 的窗口们，这些窗口以不同的 $k$ 为中心。还有$\overline{a_i}=\frac{1}{|w|}\sum_{k:i\in w_k}a_k,\overline{b_i}=\frac{1}{|w|}\sum_{k:i\in w_k}b_k$

1.3 Conclusion

不难发现，导向滤波效果的好坏几乎完全取决于导引图 $I$ 的质量，但是在遇到要大量处理图片的情况下，又不可能一张一张手工做导引图，所以一般来说导引图不是原图像自身 $I=p$；就是原图像的灰度图 $I=gray(p)$。当然这样做也有些许道理，因为灰度图正好反应了原图像的深浅程度。

当 $I=p$ 时，$a_k,b_k$ 就会退化为$$
\begin{cases}
a_k = \dfrac{\sigma^2_k}{\sigma^2_k+\epsilon}\\
b_k = (1-a_k)\mu_k
\end{cases}
$$此时：

  • 当遇到边缘地区，即方差比较大的地方 $\sigma^2_k>>\epsilon$ 时，$a_k\simeq1,b_k\simeq0$，则结果保持不变，即 $q=I$
  • 当遇到平坦的区域，即方差比较小的地方 $\sigma^2_k<<\epsilon$ 时，$a_k\simeq0,b_k\simeq\mu_k$，则结果就是对 $p$ 进行均值滤波。

也就是说在方差比较大的区域，保持值不变，在平滑区域，使用临近像素平均（也就退化为普通均值滤波）。

而 $\epsilon$ 的作用就是界定什么是变化大，什么是变化小。在窗口大小不变的情况下，随着 $\epsilon$ 的增大，滤波效果越明显。

在滤波效果上，引导滤波和双边滤波差不多，在一些细节上，引导滤波较好。引导滤波最大的优势在于，可以写出时间复杂度与窗口大小无关的算法，因此在使用大窗口处理图片时，其效率更高。
